API Documentation

This API allows users to retrieve and add data to ClipBin. It includes two primary functions: get_data (GET request) and post_data (POST request).

API Endpoints

GET

/get_data

Retrieve data from the database

Retrieve data from the database based on specific parameters.

Query Parameters

Parameter Type Required Description
id string Yes The unique identifier for the data.
name string No The name associated with the data.
pwd string No The password for authorization.
unlisted string No Filter for unlisted data. Accepted values: true or false.

Responses

200 OK - Successfully retrieved data
JSON Response 
[
  {
    "id": "URL Id",
    "name": "Sample Name", 
    "text": "Sample Text",
    "time": "Saving Time"
  }
]
400 Bad Request - Missing or incorrect parameters
Error Response 
{
  "Error": "Missing Parameters!"
}
Error Response 
{
  "Message": "To Search, enter Id not Name with unlisted!"
}
401 Unauthorized - Authentication failed
Error Response 
{
  "Error": "Incorrect Password"
}
404 Not Found - No data found
Error Response 
{
  "Error": "No Data"
}

Example Request

HTTP Request 
GET /get_data?id=1&name=Sample&pwd=example_password&unlisted=false
POST

/post_data

Add new data to the database

Add new data to the database with the provided details.

Request Parameters

Parameter Type Required Description
name string Yes The name associated with the data.
text string Yes The text content to be stored.
pwd string No The password for protecting the data.
unlisted string No Specify if the data should be unlisted. Accepted values: true or false.
remove string No Specify when the clip will be removed. Available options:
  • day - for 1 day
  • week - for 1 week
  • twoweek - for 2 weeks
  • month - for 1 month
  • half - for 6 months
  • year - for 1 year

If any other option is provided, it will be treated as never.

Responses

201 Created - Successfully added data
Success Response 
{
  "id": "url_id", 
  "Message": "Successfully added!"
}
400 Bad Request - Missing parameters or failed to add data
Error Response 
{
  "Error": "Missing Parameters!"
}
Error Response 
{
  "Error": "Couldn't add. Something went wrong."
}

Example Request

HTTP Request 
POST /post_data

Body:
{
  "name": "Sample Name",
  "text": "Sample Text",
  "pwd": "example_password", 
  "unlisted": "false",
  "remove": "half"
}

Error Codes

Status Code Description
200 OK – Request succeeded.
201 Created – Resource successfully created.
400 Bad Request – Request was invalid.
401 Unauthorized – Authentication failed.
404 Not Found – Resource not found.

Authentication

The pwd parameter is required for retrieving protected data and optional for adding data.

When a clip is password protected, you must provide the correct password to access its content.
